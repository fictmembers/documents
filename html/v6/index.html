Модуль листування


Додати модулю для листування, що забезпечить спілкування між компаніями та користувачами.

Опис будови модулю «листування»:


Для модулю листування у данні версії роботи можливі наступні функції:

  • destroy - знищенняповідомлення.
  • new - утворює нове повідомлення.
  • create - зберігає нове повідомлення у БД.
  • promo_require - захисна функція, що не дає утворювати повідомлення незалогіненому користувачу.
  • correct_company - повертає запис на поточного користувача.

Взаємодія з таблицею у БД:


Як можемо бачити, у моделі прописано, що наші листи належать до їх утворювача - компанії. Тобто у базі даних компаній до кожного листа приєднується посилання на його хозяїна через forein key. За замовчанням усі повідомлення будуть виводитися відсортованими по часу їх додаваняя, але у зворотньому порядку.

Допоміжний механізм:


Для того, щоб при написанны листа, в адресати можна було обрати кого завгодно, використовується допоміжний механізм, що проводить пошук та виводить усіх зареєстрованих користувачів.

Представлення у таблиці:


Для кожного «листа» передбачаються наступні поля:

  • text - Інформаційний блок листа.
  • recipient - Пошта для авторизації.
  • created_at - Час утворення запису.
  • updated_at - Час утворення запису.
  • task - Тема завдання.
  • company_sender - Назва компанії, що надсилає повідомлення, .
  • company_id - foreign key.
