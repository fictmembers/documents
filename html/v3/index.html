Профіль компанії


Утворити кабінет компанії з усіма належними та необхідним частинами (реєстрація, вхід, редагування та додання інформації).

Опис будови модулю «Екзаменаторів»:


    Для модулю екзаменатора у данні версії роботи можливі наступні функції:
  • show - виводить головну сторінку для поточного користувача.
  • index - виводить список усіх екзаменаторів.
  • new - утворює новий обліковий запис.
  • create - зберігає новий обліковий запис.
  • edit - дозволяє редагувати вже існуючий профіль.
  • update - дозволяє додати нову інформацію про поточного користувача модуля.
  • company_params - захисна функція для перевірки правильності введення інформації при регістрації.
  • signed_in_company - захисна функція, що не дає переглядати і змінювати незалогіненому користувачу змінювати інформацію.
  • correct_company - повертає запис на поточного користувача.

Взаємодія з таблицею у БД:

Можемо бачимо, що поточний стан компанії дозволяє утворювати зашифровані паролі для «екзаменаторів» функцією has_secure_password. При реєстрації для правильності заповнення бази, від людини будуть вимагатися наступні дані: ім’я, пошту та назву компанії. Для перевірки правильності введеної пошти використовується шаблон VALID_EMAIL_REGEX. Для кожної сесії утворюється власний токен безпеки, що зберігається у кукі.

Налаштуванн шляхів:


Для коректної роботи проекту, модуль «екзаменаторів» був доданий до ресурсів (тобто усі стандартні сторінки автоматично будуть підхоплені при виконанні додатку). Але був переписаний шлях до поточного профілю для того, щоб динамічно відображати id користувача.

Представлення у таблиці:


Для кожного «екзаменатора» передбачаються наступні поля:

  • name - Назва організації, де працює екзаменатор.
  • email - Пошта для авторизації.
  • manager - ПІБ екзаменатора.
  • created_at - Час утворення запису.
  • updated_at - Час утворення запису.
  • password_digest - зашифрований пароль.
  • remember_token - токен авторизації.
